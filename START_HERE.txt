┌────────────────────────────────────────────────────────────────────────────┐
│ 📽️ HOW IT WORKS                                                          │
└────────────────────────────────────────────────────────────────────────────┘

DATA FLOW:
  Streamlit UI (Sidebar Navigation)
        ↓
  Analytics Engine (Fraud Detection, Risk Scoring)
        ↓
  Machine Learning Models (Isolation Forest, Statistical Analysis)
        ↓
  SQLite Database (data/banksight.db) - Persistent Storage
        ↓
  Plotly Charts, DataFrames, Metrics

FRAUD DETECTION PIPELINE:
  1. Transaction arrives
  2. Features extracted (amount, merchant, device, location, time)
  3. ML model scores anomalies (0 = safe, 1 = fraud)
  4. Risk components weighted (behavioral, geographic, temporal, etc.)
  5. Final risk score calculated (0.0-1.0)
  6. Flagged if score > 0.7 threshold

┌────────────────────────────────────────────────────────────────────────────┐
│ 🛟 TROUBLESHOOTING                                                        │
└────────────────────────────────────────────────────────────────────────────┘

Issue: App won't start
Solution:
  1. Check if port 8501 is in use: netstat -ano | findstr :8501
  2. Kill process if needed: taskkill /PID <PID> /F
  3. Try again: .venv\Scripts\streamlit.exe run app.py

Issue: Sample data won't load
Solution:
  1. Refresh browser (F5)
  2. Check logs: type logs\banksight.log
  3. Click button again

Issue: Charts not showing
Solution:
  1. Update Plotly: .venv\Scripts\pip install --upgrade plotly
  2. Clear browser cache: Ctrl+Shift+Delete
  3. Refresh page

Issue: Database errors
Solution:
  1. Verify database exists: ls data/banksight.db
  2. Recreate database: .venv\Scripts\python.exe setup-sqlite.py
  3. Check logs for details
